Methodology

Understanding how we derive insights from local data.

Data Sources & Schema

FreeWorld XenoData is built entirely on local files. Our primary dataset, sightings.csv, contains fields such as timestamp, location (country, state, city, latitude, longitude), shape, duration, description and confidence metrics. A secondary file, points_of_interest.public.en, lists named locations like parks, landmarks and tunnels with coordinates and category labels. At build time we validate fields using strict schemas, derive additional attributes (like month, quarter and description length) and quarantine any invalid rows to maintain data integrity.

Processing Pipeline

All analytics are precomputed locally. We aggregate sightings by city, region, shape and time; calculate z‑scores to identify anomalies; compute correlation matrices and diversity indices; evaluate categorical associations using Cramér’s V and mutual information; cluster points using density algorithms; detect changepoints and structural breaks; and estimate short‑term forecasts. Heavy computations run in Web Workers where supported, ensuring a responsive interface. Caches keyed by filter hashes avoid redundant work.

Statistics & Models

Our analytic toolkit includes Pearson and Spearman correlations for continuous variables, Cramér’s V and φ coefficients for categorical associations, Shannon diversity indices, kernel density estimates for spatial clustering, and simple panel models to explore counts over time. We avoid p‑hacking by treating p‑values cautiously and emphasising effect sizes. Interpretive text accompanies each chart to contextualise results and note limitations (e.g., stationarity assumptions for z‑scores or potential confounders in correlation analyses).

Performance & Accessibility

To deliver a smooth investigative experience, the site targets 60fps rendering. We use canvas for heavy plots and fall back to simplified representations for dense scatter. Images are served in modern formats (AVIF/WEBP) with PNG fallbacks and include alt text. Keyboard navigation and high contrast modes are available across all sections, and motion effects respect user preferences via prefers‑reduced‑motion. No external trackers or APIs are used; your interactions remain private.
